## Decisions Made

The following seven behaviours are confirmed as the complete behavioural scope for the meta-canary QR code generator:

1. A user logs in with a password to access the application
2. A user enters a URL and a label, and generates a QR code that encodes that URL directly
3. A user views a list of previously generated QR codes (with their labels and URLs)
4. A user selects a specific QR code from the history
5. A user exports a selected QR code as a PNG file
6. A user exports a selected QR code as an SVG file
7. A user deletes a QR code from the history

This scope is confirmed as adequate for the purposes of the meta-canary.

- **The three pending architectural decisions** (persistence mechanism, hosting platform, password control approach) must be resolved in Session 4 before the coding agent brief can be finalised.
